# Contributing

Contributions that improve the accuracy, clarity, or coverage of this guide are welcome.

## What Makes a Good Contribution

- Correcting a factual error — wrong HTTP status code usage, outdated library API, broken link
- Adding a missing reference that is genuinely relevant and from a credible source
- Improving an explanation that is unclear, incomplete, or misleading
- Adding a code example that demonstrates a pattern more clearly
- Fixing a TypeScript or syntax error in a code example

A contribution is not a good fit if it adds opinion without a cited source, promotes a specific tool without addressing trade-offs, or rewrites content without a clear reason.

## Process

1. Open an issue first for anything beyond a minor fix. This allows discussion before you spend time writing.
2. Fork the repository and create a branch from `main`:
   ```bash
   git checkout -b fix/wrong-status-code-in-section-2
   git checkout -b feat/add-graphql-section
   ```
3. Make your changes. Keep pull requests focused on one change at a time.
4. Open a pull request with a clear description of what changed and why.

## Adding References

When adding a new reference, include: title, author(s), year, publisher or organization, and URL where available. Add it to both the relevant section's `## Sources` block and to `docs/13-references.md`.
